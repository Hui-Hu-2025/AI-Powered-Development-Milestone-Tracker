# 隐私和安全说明

## ⚠️ 当前数据使用情况

### 发送到 OpenAI 的数据

当前代码会将以下**敏感信息**发送到 OpenAI API：

1. **孩子真实姓名**
2. **出生日期**（可推算年龄）
3. **特殊情况描述**（可能包含医疗敏感信息）
4. **所有发育记录**：
   - 身高、体重、头围
   - 大运动、语言、精细动作发展情况
   - 睡眠、饮食情况
   - 历史记录数据

### 数据泄露风险

**高风险因素：**
- ✅ 数据通过 HTTPS 加密传输（相对安全）
- ⚠️ 数据会发送到 OpenAI 服务器进行处理
- ⚠️ OpenAI 可能用于模型训练（取决于你的 API 设置）
- ⚠️ 包含可识别个人身份的信息（姓名、出生日期）

## OpenAI 数据使用政策

### 默认情况（2024年3月前）
- OpenAI **可能**使用 API 数据来训练模型
- 数据可能被存储和用于改进服务

### 企业版/API 设置
- 可以设置数据不用于训练
- 需要检查你的 OpenAI 账户设置

### 查看和设置
1. 访问：https://platform.openai.com/settings/data-usage
2. 检查 "Data usage for training" 设置
3. 企业版可以禁用数据用于训练

## 🔒 安全建议

### 方案 1：数据脱敏（推荐）

在发送到 OpenAI 前，移除或替换敏感信息：

- ✅ 移除真实姓名，使用代号（如"孩子A"）
- ✅ 只发送年龄（月），不发送出生日期
- ✅ 移除可识别身份的特殊信息

### 方案 2：使用本地模型

- 使用本地部署的开源模型（如 Ollama、Llama）
- 数据完全不离开你的服务器
- 需要更强的硬件支持

### 方案 3：使用企业版 API

- OpenAI 企业版提供数据不用于训练的保证
- 需要付费订阅

### 方案 4：数据加密和匿名化

- 在发送前对数据进行加密或哈希处理
- 使用假名系统

## 📋 合规建议

1. **告知用户**：明确告知数据会被发送到第三方服务
2. **获得同意**：获得用户（家长）的明确同意
3. **数据最小化**：只发送必要的评估数据
4. **数据保留**：明确数据保留期限
5. **访问控制**：确保只有授权人员可以访问

## 🛡️ 最佳实践

1. **定期审查**：定期检查发送到 API 的数据
2. **日志记录**：记录所有 API 调用
3. **访问控制**：限制谁可以使用系统
4. **数据备份**：定期备份本地数据库
5. **安全更新**：保持依赖包更新

## 法律合规

根据地区不同，可能需要遵守：
- **GDPR**（欧盟）：需要明确同意和数据可携带权
- **HIPAA**（美国医疗）：医疗数据有特殊要求
- **个人信息保护法**（中国）：需要明确告知和同意
